埋点方案

代码埋点

也称为手动埋点，通过引入数据统计服务商的sdk，自定义事件，如果需要精细化埋点，工程量大，后期维护成本高

比如百度统计、友盟

可视化埋点

通过可视化交互的手段，代替上述的代码埋点。将业务代码和埋点代码分离，提供一个可视化交互的页面，输入为业务代码，通过这个可视化系统，可以在业务代码中自定义的增加埋点事件等等，最后输出的代码耦合了业务代码和埋点代码

缺点：可以埋点的控件有限，不能手动定制、成本高

比如腾讯的MTA，支持npm安装

无埋点

无埋点则是前端自动采集全部事件，上报埋点数据，由后端来过滤和计算出有用的数据。

优点：前端只要一次加载埋点脚本

缺点：流量和采集的数据过于庞大，服务器性能压力山大

Quick Start

从URL到页面展示过程

URL->DNS寻址->IP层级查找（从右到左）->Request start ->服务器查找到资源->Response start->页面渲染

页面渲染过程

DOM树生成（所有节点）->资源下载->render树生成（可见节点）->逐级解析DOM树，展示页面

过程中js要放在最后，因为js引擎涉及到对DOM和CSS的操作，会阻塞渲染。script有两个属性，分别是async和defer，两个虽然都是异步加载，但defer是在整个DOM渲染完才执行，而async 则是一个乱序执行的主，反正对它来说脚本的加载和执行是紧紧挨着的，所以不管你声明的顺序如何，只要它加载完了就会立刻执行。如果没有使用这两个属性，那么浏览器会立即加载并执行指定的脚本，读到即执行

ES6 Module

Posted on 2019-01-08 | In js

在ES6之前，社区制定了一些模块加载方案，最主要的有CommonJS和AMD两种。前者用于服务器，后者用于浏览器。ES6在语言标准的层面上，实现了模块功能，完全可以取代CommonJS和AMD规范，成为浏览器和服务器通用的模块解决方案。

Quick Start

特点

ES6 模块的设计思想是尽量的静态化，使得编译时就能确定模块的依赖关系，以及输入和输出的变量。CommonJS 和 AMD 模块，都只能在运行时确定这些东西。比如，CommonJS模块就是对象，输入时必须查找对象属性。

ES6 模块不是对象，而是通过export命令显式指定输出的代码，再通过import命令输入。

上面代码的实质是从fs模块加载3个方法，其他方法不加载。这种加载称为“编译时加载”或者静态加载，即 ES6 可以在编译时就完成模块加载，效率要比 CommonJS 模块的加载方式高。

模块功能主要由两个命令构成：export和import。

export

export命令用于规定模块的对外接口。

一个模块就是一个独立的文件。该文件内部的所有变量、方法，外部无法获取。

另外，export语句输出的接口，与其对应的值是动态绑定关系，即通过该接口，可以取到模块内部实时的值

最后，export命令可以出现在模块的任何位置，只要处于模块顶层就可以。如果处于块级作用域内，就会报错，下一节的import命令也是如此。这是因为处于条件代码块之中，就没法做静态优化了，违背了 ES6 模块的设计初衷。

import

import命令用于输入其他模块提供的功能；import命令输入的变量都是只读的，但如果是对象，可以对对象的属性进行修改；

import后面的from指定模块文件的位置，可以是相对路径，也可以是绝对路径，.js后缀可以省略

注意，import命令具有提升效果，会提升到整个模块的头部，首先执行

import语句会执行所加载的模块

多次重复执行同一句import语句，那么只会执行一次，而不会执行多次

模块的整体加载

export default

export default命令用于指定模块的默认输出。显然，一个模块只能有一个默认输出，因此export default命令只能使用一次

注意：第一组是使用export default时，对应的import语句不需要使用大括号；第二组是不使用export default时，对应的import语句需要使用大括号

import()函数

由于import命令是编译时加载，无法取代require()这种运行时加载，所以引入了import()函数，需要传的参数和import命令一样

import()函数返回的是一个Promise对象，异步加载

另外，import()函数与所加载的模块没有静态连接关系

async await简单使用

Posted on 2019-01-07 | In js

async-await用来处理异步请求，解决“回调地狱”问题，ES6也对此作出了标准，就来看看它的基本使用。

Quick Start

async语法

async做为一个关键词，放在函数前面，就表示函数是一个异步函数。调用方式和正常函数一样

输出结果为：我先执行了 Promise:{:’hellow world’}

可以看到async返回的是一个Promise对象，可以通过then方法，获取到返回值

输出结果为：我先执行了 hello world

async内部实现原理

如果async函数返回一个值，当调用该函数时，内部会调用Promise.resolve()方法，把结果转换为一个Promise对象返回；如果内部抛出错误，那么调用Promise.reject()方法，返回一个Promise对象

await简单使用

await必须放到async函数中，后面可以跟任何表达式，但常用的是一个返回promise对象的表达式

会先输出Promise对象的状态为Pending，2s后输出6

场景

在一个接口级联的情况下这种处理方式更简单。比如，当用户输入电话号码后，先查找这个电话号码所在的省和市，然后再根据省和市，找到可能充值的面值，进行展示
